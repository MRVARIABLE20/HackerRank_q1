SECTION 1: COMPLETE RAG TAXONOMY (All Known Types as of 2026)
=============================================================================

TIER 1 — FOUNDATIONAL
  01. Naive RAG           | The original fixed retrieve-once pipeline
  02. Vector-less RAG     | BM25/TF-IDF keyword retrieval, no embeddings
  03. SQL RAG             | Natural language → SQL → structured DB query

TIER 2 — PIPELINE IMPROVEMENTS
  04. Advanced RAG        | Pre/post retrieval enhancements (query rewriting, reranking)
  05. Hybrid RAG          | Dense (vector) + Sparse (BM25) + Reciprocal Rank Fusion
  06. Modular RAG         | Plug-and-play interchangeable components
  07. RAG-Fusion          | Multi-query generation + RRF merging
  08. Speculative RAG     | Small specialist drafts → large model verifies
  09. Multi-hop RAG       | Iterative retrieval for multi-document reasoning

TIER 3 — SELF-CORRECTING
  10. Self-RAG            | LLM generates reflection tokens to control retrieval
  11. Corrective RAG      | External evaluator scores retrieval quality + web fallback
  12. Adaptive RAG        | Query complexity classifier routes to cheapest sufficient strategy
  13. Bidirectional RAG   | Retrieves from query→doc AND doc→query directions

TIER 4 — STRUCTURED / GRAPH
  14. Graph RAG           | Knowledge graph with entity nodes and relationship edges
  15. Hybrid GraphRAG     | Graph traversal + vector search combined
  16. mKG-RAG             | Multimodal knowledge graph (text + image entities)

TIER 5 — AGENTIC
  17. Agentic RAG         | LLM agent controls retrieval in a dynamic loop
  18. Multi-Agent RAG     | Multiple specialized agents collaborate
  19. A-RAG               | Hierarchical retrieval interfaces at scale

TIER 6 — MULTIMODAL
  20. Multimodal RAG      | Text + images + tables + audio + video
  21. VisRAG              | Purely visual — document pages as images (ICLR 2025)
  22. BayesRAG            | Probabilistic multi-modal evidence fusion

TIER 7 — SPECIALIZED
  23. Long-Context RAG    | Stuffs large context into 128k–1M token models
  24. Security RAG        | Access control, PII filtering, audit trails
  25. Multilingual RAG    | Cross-language retrieval and generation

SECTION 3: DECISION GUIDE — WHICH RAG TO USE?
=============================================================================

QUESTION 1: What kind of documents do you have?
  Unstructured text only → Naive, Hybrid, or Advanced RAG
  Tables/CSVs/databases → SQL RAG
  Images, PDFs with charts → Multimodal RAG
  All of the above → Modular RAG

QUESTION 2: How complex are typical user queries?
  Simple factual ("What is our policy on X?") → Naive RAG, Adaptive RAG
  Moderate one-hop ("Summarize all incidents in April") → Advanced or Hybrid RAG
  Complex multi-hop ("Which customers violate our retention policy?") → Multi-hop or Graph RAG
  Unknown mix → Adaptive RAG (routes automatically)

QUESTION 3: How important is accuracy vs speed?
  Speed first → Naive RAG or Speculative RAG
  Accuracy first → Self-RAG or Corrective RAG
  Balanced → Adaptive RAG or Hybrid RAG

QUESTION 4: Can you fine-tune the model?
  Yes → Self-RAG (best for hallucination control)
  No → Corrective RAG (plug-and-play evaluator)

QUESTION 5: What is your infrastructure budget?
  Minimal → Naive RAG or Vector-less RAG
  Moderate → Hybrid RAG + Reranker
  High → Graph RAG or Agentic RAG
  Enterprise → Multi-Agent RAG

QUESTION 6: Do queries span entity relationships?
  Yes → Graph RAG
  No → Hybrid or Advanced RAG
